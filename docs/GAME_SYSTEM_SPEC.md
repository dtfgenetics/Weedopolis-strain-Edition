# Weedopolis: Strain City Edition — Full Game System Spec

## Production stance

The first visual attempt was wrong because it treated the project like a decorative poster. Weedopolis must be built as a real board-game production system.

From this point forward, every asset must connect to one of these systems:

1. Board spaces
2. Deed cards
3. Draw cards
4. Money
5. Player tokens
6. Upgrade pieces
7. Rulebook
8. Box and print packaging
9. Digital game data

## Game identity

- **Name:** Weedopolis
- **Edition:** Strain City Edition
- **Type:** Cannabis-themed property-trading board game
- **Player feel:** Buy strains, collect sets, build Grow Tents, upgrade to Dispensaries, bankrupt friends.
- **Tone:** Funny but clean, premium, readable, easy to play.
- **Visual style:** Vintage board game + cannabis city map + dispensary menu.

## Board structure

The board has exactly **40 spaces**.

| Space family | Count | Purpose |
|---|---:|---|
| Corners | 4 | Major turn events and pacing spaces |
| Strain properties | 22 | Main buy/rent/build spaces |
| Category spaces | 4 | Railroad-style set: Indica, Sativa, Hybrid, Autoflower |
| Card spaces | 6 | Draw High Chance or Community Stash |
| Utilities | 2 | Grow Lights and Water Works |
| Taxes/fees | 2 | 420 Tax and Lab Testing Fee |

## Asset creation rule

Do not create a final board image first. The correct asset order is:

1. Board-space specifications
2. Deed card values
3. Card deck text
4. Blank board geometry
5. Four row templates
6. Individual icons
7. Individual board spaces
8. Row overlays
9. Final board assembly
10. Print exports

## Asset quality standard

A usable asset must be one of:

- Editable SVG
- Transparent PNG
- 300 DPI PDF/PNG
- CSV data
- JSON data
- Markdown specification

A pretty image with inaccurate text is not a production asset.
